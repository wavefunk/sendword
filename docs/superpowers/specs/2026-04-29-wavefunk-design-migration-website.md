# Website Wave Funk Design System Migration

## Goal

Migrate sendword's Eigen-based website (landing page + docs) from a custom 912-line `style.css` to the Wave Funk design system, following the pattern established by substrukt's website.

## Scope

Style and template migration only. All content (features.yaml, docs.yaml, categories.yaml, nav.yaml, site.toml) stays unchanged.

## Architecture

The Wave Funk design system is a pure-CSS system with 6 layers (`01-tokens` through `06-marketing`) bundled via `wavefunk.css`. Each product adds a tiny override file for accent color. Templates use two class prefixes: `mk-*` for marketing pages and `wf-*` for product/docs pages.

### CSS

**Add:** `static/css/wavefunk/` ‚Äî full copy of `../design/css/` including fonts  
**Add:** `static/css/sendword.css` ‚Äî accent color overrides only:

```css
:root {
  --accent: #cba6f7;
  --accent-ink: #000000;
}

[data-mode="light"] {
  --accent: #8839ef;
  --accent-ink: #ffffff;
}
```

**Delete:** `static/css/style.css`

### Template Hierarchy

```
_base.html                  ‚Üí HTML shell, loads wavefunk.css + sendword.css, HTMX
  _marketing.html           ‚Üí mk-wrap wrapper, nav + footer partials (NEW)
    index.html              ‚Üí Landing page using mk-hero, mk-sect, mk-feat, mk-step
  _docs.html                ‚Üí wf-docs-shell, sidebar + content + toc
    docs/index.html          ‚Üí Docs landing with category cards
    docs/[doc].html          ‚Üí Individual doc with wf-prose markdown
  404.html                   ‚Üí Simple error page
```

### Templates ‚Äî What Changes

**`_base.html`** ‚Äî Replace font imports (Outfit/Space Mono ‚Üí Martian Grotesk/Mono from wavefunk fonts), replace CSS link (`style.css` ‚Üí `wavefunk/wavefunk.css` + `sendword.css`), keep HTMX, remove all custom JS (copy buttons, sidebar toggle ‚Äî design system handles these or they move to specific templates). Set `data-mode="dark"` on html element.

**`_marketing.html`** (new) ‚Äî Wraps content in `mk-wrap` div, includes `_partials/nav.html` and `_partials/footer.html`. Provides `content` block.

**`index.html`** ‚Äî Extends `_marketing.html` instead of `_base.html`. Remap classes:
- `.hero` ‚Üí `mk-hero` with `mk-hero-inner`, `mk-hero-eyebrow` (for the badge text)
- `.hero-title` ‚Üí `mk-hero h1` with `<em>` for accent on "word"
- `.hero-tagline` ‚Üí `mk-hero p`
- `.hero-actions` ‚Üí `mk-hero-cta` with `wf-btn lg primary` / `wf-btn lg`
- `.hero-demo` ‚Üí `mk-code` section (webhook ‚Üí command demo)
- `.features-grid` ‚Üí `mk-features` with `mk-feat` cards (`mk-feat-num`, `mk-feat-t`, `mk-feat-b`)
- `.steps` ‚Üí `mk-steps` with `mk-step` items
- `.section-config` ‚Üí `mk-sect` with `mk-code` for config showcase
- `.cta` ‚Üí `mk-sect` with shell-line install command

**`_docs.html`** ‚Äî Replace custom 2-column layout with `wf-docs-shell` (sidebar + content). Content area uses `wf-docs-content` with `wf-prose` for markdown. Include sidebar partial in `wf-docs-side`.

**`_partials/nav.html`** ‚Äî Replace custom sticky header with `wf-mnav`. Logo uses sendword icon (accent-colored "S" square) + wordmark. Nav items from nav.yaml.

**`_partials/footer.html`** ‚Äî Replace simple text footer with `mk-foot` 4-column grid: brand column, resources (docs, quickstart), project (GitHub, license), operations (backup, API). Add `mk-colophon` with Wave Funk badge and copyright.

**`_partials/sidebar.html`** ‚Äî Replace custom `<details>` categories with `wf-docs-side` sections. Each category becomes a section header with links. HTMX attributes (`hx-get`, `hx-target`, `hx-push-url`) for fragment-based navigation. Active link detection via `is-active` class.

**`docs/index.html`** ‚Äî Restyle category cards using design system classes. Keep category iteration from categories.yaml.

**`docs/[doc].html`** ‚Äî Minimal changes ‚Äî wrap content in `wf-prose`, ensure markdown filter output renders correctly with design system typography.

**`404.html`** ‚Äî Simplify using design system typography. Keep "message not delivered" flavor text.

### What Stays Unchanged

- `site.toml` ‚Äî Build config, SEO, sitemap settings
- `_data/features.yaml` ‚Äî Feature card content
- `_data/docs.yaml` ‚Äî All documentation content
- `_data/categories.yaml` ‚Äî Doc category definitions
- `_data/nav.yaml` ‚Äî Navigation links
- `dist/` ‚Äî Regenerated by `eigen build`
